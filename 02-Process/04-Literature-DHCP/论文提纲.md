### **Abstract**

The secure bootstrapping of producers in Named Data Networking (NDN) is a foundational process for establishing trust in the network. However, existing security bootstrapping schemes predominantly rely on Out-of-Band (OOB) communication, which is not only operationally cumbersome but also fundamentally constrained to one-hop proximity. This limitation severely hinders the deployment of NDN in scenarios requiring remote or dynamic device onboarding. **This paper investigates the critical research question: How can a new producer securely disseminate its configuration information to its designated registering node when no routing state for its prefix exists in the network**?​ To address this challenge, we propose a novel bootstrapping method that leverages NDN's native communication paradigms. **The cornerstone of our solution is a routing-driven, producer-initiated security information push mechanism**​. 

As illustrated in Figure 1, the new producer encapsulates its security configuration into a specific Interest packet. This Interest is designed to be forwarded towards the registering node based on a pre-configured, reachable prefix (e.g., the registering node's own prefix or a dedicated bootstrapping prefix). Upon receiving this Interest, the registering node can then fetch the producer's security information via a corresponding Data packet. This approach eliminates the dependency on OOB channels and one-hop physical limitations, thereby enabling truly remote bootstrapping. Preliminary analysis indicates that our method provides a scalable and NDN-native alternative to existing bootstrapping procedures.

### **Question & Challenge**

**1. 研究问题与研究目标**

本研究旨在解决的核心科学问题是：**在命名数据网络（Named Data Networking, NDN）中，当新加入的生产者其名字前缀的路由信息尚未在网络中传播时，如何利用NDN原生的通信机制，实现该生产者与远程注册节点之间安全配置信息的可靠、安全交换？**

本研究的目标是设计并验证一种全新的安全引导协议，该协议能够明确回答上述问题，其核心是使新生产者能够主动地、安全地向网络内部的注册节点“宣告”自身的存在与安全凭证，而无需依赖任何带外（Out-of-Band, OOB）通道或受限于一跳的物理距离。

**2. 问题的重要性与挑战性**

**重要性**​ 体现在以下两个层面：

- **基础安全性保障**：​安全引导是NDN架构中信任链建立的起点。生产者只有成功地向网络验证其身份并注册其前缀，后续基于名字的数据通信安全（如签名验证）才有意义。一个脆弱或不可用的引导过程会直接危及整个网络的安全根基。
- **应用场景普适性**：现有基于OOB和一跳邻接的引导方法（如物理接触、二维码扫描等）将NDN的应用范围限制在了小型、静态的物联网或局域网场景。对于广域分布式系统、远程设备部署（如远程工业传感器、家庭网关）或动态移动节点，这些方法显然不适用。因此，解决远程引导问题对于释放NDN在更广阔互联网应用场景中的潜力至关重要。

**挑战性**​ 则源于NDN架构的基本原理与引导阶段特殊性的内在矛盾：

- **路由与通信的“鸡与蛋”问题**：​NDN通信的基本模式是消费者通过名字前缀来请求数据。然而，在引导阶段，新生产者的名字前缀对于网络中的路由器是未知的，因此无法通过常规路由机制将针对该前缀的Interest包送达生产者。反过来，生产者也无法通过标准方式被“发现”。如何在不依赖目标前缀路由状态的前提下，建立初始通信通道是本问题最大的挑战。
    
- **安全信任的初始化悖论**：​安全引导需要在互不信任或未建立信任的实体间交换安全敏感信息。在OOB方式中，物理接触提供了初始信任锚。但在纯带内（In-Band）远程引导中，如何确保引导信息本身（如公钥）的完整性和真实性，防止中间人攻击或信息篡改，是一个严峻的安全挑战。初始信任的建立机制需要精巧的设计。
- **可扩展性与通用性要求**：​解决方案不应引入复杂的中间代理或改变NDN核心转发范式。它需要尽可能地利用现有NDN协议元素（如Interest/Data交换、选择策略等），保证方案的轻量化和与未来NDN网络发展的兼容性。

**综上所述，实现远程、安全的NDN生产者引导是一个兼具基础重要性与高度挑战性的研究问题，其解决将直接推动NDN从实验室原型走向大规模实际应用。**
